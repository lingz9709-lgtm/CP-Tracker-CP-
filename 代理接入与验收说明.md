# 小程序 DeepSeek 代理接入与验收说明

## 1. 架构
- 调用链路：`小程序 -> 你的后端 /api/chat -> DeepSeek /chat/completions`
- 目的：避免小程序直连外部模型域名导致“request 合法域名”报错，同时防止 `DEEPSEEK_API_KEY` 泄露到前端。

## 2. 后端落地
- 已实现接口：`POST /api/chat`
- 文件：`server/api.js`
- 关键能力：
  - 从环境变量读取 `DEEPSEEK_API_KEY`
  - 上游超时控制：`10s`
  - 请求体校验：`messages` 必填、空消息剔除
  - 非 200 返回统一结构：`{ code, msg }`
  - 成功返回统一结构：`{ code: 0, msg: "ok", data }`

## 3. 小程序端落地
- 配置文件：`小程序版/miniprogram/config.js`
  - `API_BASE_URL`：填写你的后端 HTTPS 域名，例如 `https://api.example.com`
  - `CHAT_PROXY_PATH`：默认 `/api/chat`
- 封装文件：
  - `小程序版/miniprogram/utils/http.js`（统一 `baseURL` + 请求/响应拦截器）
  - `小程序版/miniprogram/utils/deepseek.js`（统一聊天请求）

## 4. 微信后台配置（必须）
- 微信公众平台 -> 开发管理 -> 开发设置 -> 服务器域名
- 在 `request 合法域名` 中添加：`https://你的后端域名`
- 微信开发者工具 -> 详情 -> 域名信息，确认该域名已生效

## 5. 验收步骤
1. 启动后端并确认健康检查：`GET /api/health`
2. 小程序触发 AI 润色/生成报告
3. 在开发者工具 Network 面板确认请求为：`https://你的域名/api/chat`
4. 页面不再出现“未配置 request 合法域名”
5. 后端日志有对应 `/api/chat` 请求记录
